# SLAM 前端代码重构说明文档

## 一、重构概述

本次重构在**不改变原有业务逻辑**的前提下，对 SLAM 前端系统的 12 个源文件进行了全面的可读性优化。

### 重构原则
1. **命名优化**：变量和函数命名更具自解释性，避免缩写
2. **卫语句模式**：将嵌套 if-else 改为提前返回，减少嵌套层级
3. **单一职责**：长函数拆分为功能单一的小函数，主函数像读故事一样清晰
4. **中文注释**：为关键步骤添加清晰的中文注释
5. **代码分组**：按功能模块分组，使用分隔符提高可读性

### 2.2 新增拆分函数说明

#### 光流追踪模块（3个）

| 函数名 | 职责 |
|--------|------|
| `PerformBidirectionalOpticalFlow` | 执行正向+反向LK金字塔光流追踪，得到初始追踪结果 |
| `FilterValidTrackedPoints` | 过滤追踪失败的点和超出图像边界的点 |
| `FilterWithEpipolarConstraint` | 使用对极约束+RANSAC剔除误匹配点 |

**主函数 `TrackFeaturesWithOpticalFlow` 流程**：
```
1. 卫语句：上一帧无特征点直接返回
2. 步骤1：执行双向光流追踪
3. 步骤2：过滤追踪失败和出界的点
4. 步骤3：对极约束RANSAC剔除误匹配
5. 步骤4：更新当前帧特征数据
```

#### PnP位姿估计模块（4个）

| 函数名 | 职责 |
|--------|------|
| `Collect3D2DCorrespondences` | 收集已三角化的3D地图点与2D特征点的对应关系 |
| `BuildCameraIntrinsics` | 构建相机内参矩阵和畸变系数 |
| `ConvertCvToEigenPose` | 将OpenCV的旋转向量和平移向量转换为Eigen位姿 |
| `FilterFeaturesByPnPInliers` | 根据PnP内点结果过滤特征，并更新地图点的内/外点状态 |

**主函数 `EstimatePoseWithPnP` 流程**：
```
1. 卫语句：无特征点或无地图点直接返回
2. 步骤1：收集3D-2D对应点对
3. 卫语句：3D点不足4个直接返回
4. 步骤2：构建相机内参
5. 步骤3：执行PnP+RANSAC求解位姿
6. 卫语句：求解失败或内点不足直接返回
7. 步骤4：转换为Eigen位姿格式
8. 步骤5：根据内点结果过滤特征点
```

#### 三角化模块（9个）

| 函数名 | 职责 |
|--------|------|
| `PerformStereoMatching` | 执行双目立体匹配（基于光流法），得到右目对应点 |
| `FilterByEpipolarLineConstraint` | 使用极线约束过滤双目误匹配点 |
| `UndistortFeaturePoints` | 对左右目特征点进行去畸变处理 |
| `CalculateCameraExtrinsics` | 计算相机外参和基线参数 |
| `TriangulateSinglePoint` | 对单个特征点执行SVD三角化求解 |
| `CheckDisparityValidity` | 检查视差是否足够大（太小则深度不可靠） |
| `CheckSvdSingularRatio` | 检查SVD奇异值比值（判断解的稳定性） |
| `CheckDepthValidity` | 检查深度是否在可靠范围内 |
| `CheckReprojectionError` | 检查重投影误差是否在阈值内 |

**主函数 `TriangulateNewMapPoints` 流程**：
```
1. 卫语句：无特征点或右目为空直接返回
2. 步骤1：双目立体匹配（光流法）
3. 步骤2：极线约束过滤误匹配
4. 步骤3：特征点去畸变
5. 步骤4：计算相机外参和基线
6. 步骤5：逐点三角化并通过4重质量校验
```

#### 特征管理模块

| 函数名 | 职责 |
|--------|------|
| `AddNewFeatures` | 在特征稀疏区域补充新的角点，保持特征均匀分布 |
| `CreateFeatureMask` | 创建特征掩码，在已有特征周围留出禁区 |
| `UpdatePreviousStatus` | 更新上一帧的特征状态（为下一帧追踪做准备） |
| `IsPointInsideBorder` | 判断点是否在图像边界内 |
| `CalculatePixelDistance` | 计算两个像素点之间的欧氏距离 |

| 函数名 | 职责 |
|--------|------|

| `BackendOptimizationLoop` | 后端优化主循环线程 |

**主函数 `ProcessStereoFrame` 流程**：
```
1. 步骤1：图像预处理（灰度 + CLAHE）
2. 步骤2：构建可视化图像
3. 读取当前位姿
4. 分支A：第一帧系统初始化
5. 分支B：正常帧间追踪
   - 步骤B1：稀疏光流帧间追踪
   - 步骤B2：PnP求解位姿
   - 步骤B3：刷新特征点（均匀分布）
   - 步骤B4：关键帧判定
   - 步骤B5：双目三角化生成新地图点
   - 步骤B6：插入关键帧并触发后端
   - 步骤B7：PnP失败降级处理
   - 步骤B8：收集关键帧位置
   - 步骤B9：绘制特征点
   - 步骤B10：计算特征速度
   - 步骤B11：更新上一帧状态
```

## 九、注意事项

1. **业务逻辑不变**：所有重构仅涉及代码结构和命名，算法逻辑完全保持一致
2. **头文件与实现同步**：修改.h后已同步修改.cpp
3. **命名规范统一**：所有文件遵循相同的命名规范（驼峰命名、m前缀成员变量）
4. **卫语句模式**：尽可能将嵌套if-else改为提前返回，提高可读性
5. **单一职责**：每个函数只做一件事，主函数负责流程编排
